关于分布式的一些认识

分布式系统定义

  • 多节点
  • 消息通信
  • 不共享内存 （这里很像Go的Channel）

三个概念

  • 分布式系统：功能分割，微服务，一个系统处理一个功能

  • 集群：多个系统处理的都是一个功能

  • 分布式计算（Gearman）：并行计算

分布式SESSION（会话控制）

分布式多台主机情况下，IP哈希使得用户可以一直访问一台主机，那么如果主机挂掉，也是有隐患的，想要在分布式系统保存session，可以使用专门一个主机（服务）去处理session

水平扩展和垂直扩展

水平扩展指的是加机器，构造集群，垂直扩展指的是拆分服务，微服务架构

对于SpringCloud套件基本功能的认识

  • Spring Eureka Server : 服务发现服务器，用于发现服务，为其他（例如消费者）提供host 也就是说可以直接用microservice的名字进行调用 多个Server集群replicat可以实现高可用热备

  • Spring Ribbon ： 负载均衡，可以以不同的模式进行对ServiceProvider提供的服务进行负载均衡式的调用

  • Spring Feign ： 另一种负载均衡，Ribbon基于resttemplate调用 多参数的时候程序不好写，Feign提供声明式调用，更加方便

  • Spring Hystrix ： 断路器，对出现故障的事例进行隔离，防止雪崩效应

  • Spring Zuul ： 用于构建微服务网关

  • Spring Config ：基于Git的统一管理微服务配置的组件

  • Spring Sleuth ： Spring的微服务耿总组件

  • SpringBoot ： 用于撰写服务提供者
