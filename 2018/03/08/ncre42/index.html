ncre4 网络工程师 day2

操作系统

  • 计算机整体设计结构 整体结构 层次结构 微内核结构(客户机/服务器结构)

  • 用户程序在用户态下使用特权指令而引起的中断是 访管中断 而 溢出中断属于异常

  • 指令错属于CPU错误 不属于IO中断 设备出错 数据传送完毕 键盘输入是可以的

  • Linux支持的状态：运行状态， 可中断睡眠状态， 不可中断睡眠状态， 暂停状态， 不可交互睡眠状态， 死亡状态， 僵死状态， 僵死撤销状态

  • Fork之后后面的不执行

  • 新创建的进程进入就绪队列不会引起新进程的调度

  • 管程是实现进程同步的有效工具 通常用来管理资源 可以提升代码可读性

  • 动态地址映射方式向内存装入程序的时候 地址转换发生在每条命令执行的时刻

  • 加快内存分配 应该按空闲区大小递增顺序排列

  • 页式存储管理方案 逻辑地址32bit 内存分块pow(2, 10) 用户程序分成 pow(2, (32 - 10))页

  • 虚拟存储空间受到计算机位宽的限制

  • 打开文件的时候所做的工作：检查文件以及对应的控制块是否已经调入内存 检查操作合法性 返回给用户一个文件描述符

  • 每个索引可以放100个地址 一级可以放100个 二级可以放100 * 100个 则全加起来就是10 + 100 + 10000

  • 影响磁盘读写性能的 寻道时间main 旋转延时 传输时间

  • 设备按信息交换单位进行分类 可以分为块设备和字符设备

  • 缓冲池着重考虑进程同步

  • 进程饥饿 进程的优先级较低而长时间得不到调度

  • 银行家算法是对死锁的避免

  • 死锁的定义 当且仅当当前状态的资源分配图是不可完全化简的

  • 进程的必要组成成分是 进程控制块 数据 指令代码

  • 每个线程拥有独立的栈， 不拥有独立的地址空间

  • PV问题：P申请过资源 资源为1 且 -1 PV也就是rc = rc + 1之类的 PV原语不可以改变顺序

  • 操作系统存储管理方法，存在碎片的是动态分区 段氏分区

  • 虚拟页式存储管理系统中，每个页表项必须包含的 页框号 有效位 修改位 访问位

  • 为了保证文件系统文件安全，可以采用定时备份 设置文件访问控制列表

  • FAT 是指Windows的File Allocation Table 文件分配表 使用十六位表示蹙号 文件物理结构是链接结构

  • IO软件的层次结构 用户应用层 设备独立层 设备驱动层 中断处理层

  • 计算机死锁 进程推进不当 资源分配不当

计算机网络

  • 801.11ae是万兆以太网 801.11z是1G 802.11ac 54mbps 802.11b 11mbps

  • 以太网中前导码字段长度 56位 7比特

  • 以太网帧长 64-1518B

  • IP互联网中主机使用同一的物理地址表示方法是错误的

  • IP数据报中 如果偏移字段值为十进制数100 那么位置偏移量为800字节

  • P2P网络的主要结构类型分为集中式 分布式结构化 分布式非结构化 混合式

  • DNS资源种类 MX 交换机 A 主机 SOA 授权开始 CNAME 域名、别名

  • FTP模式 主动模式和被动模式

  • 电子邮件系统使用base64编码的主要目的是将二进制码转化为ascii码

  • Telnet中使用nvt的目的是屏蔽不同用户终端格式差异

  • mSN使用MSNP协议 AOL采用OSCAR协议 Google Talk采用XMPP XMPP采用简单客户端 体系结构和Email类似 采用分布式非P2P协议

  • BT下载 tracker存储用户信息 最早的P2P 共享系统是Napster

  • SNMP协议已经超越了TCPIP协议 再上一层 第一部协议并没有充分考虑安全性 MIB集中存储于代理软件 收集数据采用中断和轮询

  • ElGamal基于椭圆曲线的公钥密码体制 生成密文长度是原文的2倍

  • X.509是由ITU和IETF共同制定的协议 顺序号是证书的唯一标识 发行者名用DN格式的[RFC 2222]标识

  • Linux不是Unix的一个变种 由编程爱好者维护

  • 无线局域网可以采用跳频扩频通信技术

  • Gigabit以太网 采用和以太网一样的帧格式 也可以使用双绞线进行连接

  • RIP采用向量-距离算法、OSPF采用链路-状态算法 OSPF比RIP收敛快 RIP比OSPF配置算法简单 RIP中的距离以跳数计算

  • 提高域名解析效率的方法 从本地域名服务器开始解析 在域名服务器中使用高速缓存 在客户机使用高速缓存

  • VoIP 网络电话 中的网守 是网络中消息控制中心 可以用来确定网关地址 可以和网关结合在一起 可以进行计费管理

  • 主动攻击 重放 篡改 拒绝服务 伪装 被动攻击 监控 流量分析
